# 06 - UI组件系统

## 1. UI架构

### 1.1 技术栈

- **框架**: React + Ink（定制版终端UI框架，位于 `src/ink.js`）
- **编译产物**: 所有源码均为 React Compiler 编译后输出，使用 `import { c as _c } from "react/compiler-runtime"` 实现自动缓存/记忆化
- **运行时**: Bun 1.3.5+，Node.js 24+
- **版本标识**: `999.0.0-restored`（从 source map 还原的代码树）

### 1.2 组件层级结构

### 1.3 上下文系统（Context Providers）

| Context | 位置 | 用途 |
|---------|------|------|
| `ThemeContext` | `design-system/ThemeProvider.tsx` | 主题状态管理（dark/light/auto） |
| `TextHoverColorContext` | `design-system/ThemedText.tsx` | 子树中无色ThemedText的悬停色 |

| `TabsContext` | `design-system/Tabs.tsx` | 标签页头部聚焦管理 |

### 1.4 渲染管线

```
终端事件 → Ink useInput/useTerminalSize
    ↓
React组件渲染（React Compiler自动memo化）
    ↓
Ink Box/Text 组件 → ANSI转义序列生成
    ↓
终端输出（字符单元格矩阵）
```

渲染关键特征：
- Ink 使用 **alternate screen buffer** 实现全屏模式
- 通过 **OSC 11** 监听终端主题变化（auto模式）
- **虚拟滚动** 仅渲染可见区域内的消息（`VirtualMessageList`）
- 状态栏使用 **防抖更新** + **ref追踪** + **AbortController** 避免闪烁

## 3. 核心组件详解

### 3.1 FullscreenLayout（全屏布局）

**路径**: `src/components/FullscreenLayout.tsx`

核心 REPL 布局组件，管理：
- `ScrollBox` 滚动容器作为主内容区
- 粘性 prompt 头部（sticky header），在滚动时保持可见
- `NewMessagesPill` 新消息提示胶囊
- 模态覆盖层（modal overlay），用于对话框/选择器
- 全屏模式使用 alternate screen buffer，鼠标跟踪

### 3.2 TextInput / BaseTextInput（文本输入）

**路径**: `src/components/TextInput.tsx`, `src/components/BaseTextInput.tsx`

**TextInput 特性**:
- 语音模式支持：录制时使用 Unicode 方块字符绘制迷你波形光标
  - 字符集：`' \u2581\u2582\u2583\u2584\u2585\u2586\u2587\u2588'`（BARS常量）
  - EMA平滑：`SMOOTH = 0.7`
  - 色相→RGB颜色动画
- 剪贴板图片提示
- 通过 `useTextInput` hook 委托给 `BaseTextInput`

**BaseTextInput 特性**:
- 光标定位：通过 `useDeclaredCursor` hook
- 粘贴处理：`usePasteHandler` hook
- 占位符渲染：`renderPlaceholder`
- 文本高亮：带视口窗口的语法高亮
- Slash命令参数提示显示

**Props**:
```typescript
{
  value: string;
  onChange: (value: string) => void;
  onSubmit: (value: string) => void;
  placeholder?: string;
  disabled?: boolean;
  // 语音模式相关
  isRecording?: boolean;
  audioLevel?: number;
  // 命令提示
  argumentHints?: ArgumentHint[];
}
```

### 3.3 Spinner（加载指示器）

**路径**: `src/components/Spinner.tsx`（562行）

两种变体：

**BriefSpinner**: 简短旋转指示器，用于状态行内联显示
**SpinnerWithVerbInner**: 带动词的旋转指示器

**关键特性**:
- **Thinking状态追踪**: 最小显示2秒，避免闪烁
- **Teammate spinner tree**: 多Agent协作时的树形加载指示
- **Token预算显示**: 显示当前token使用量和预算
- **动词轮转**: 循环显示状态描述文本
- **CLI活动追踪**: 区分CLI活动和空闲状态
- **动画帧**: 使用 `useAnimationFrame` 实现平滑旋转动画

**导出**: `SpinnerWithVerb`, `BriefSpinner`, `BriefIdleStatus`, `Spinner`

## 4. 终端渲染

### 4.1 颜色系统

**主题颜色**（通过 ThemeProvider 管理）:
```typescript
type Theme = {
  primary: string;      // 主色调
  secondary: string;    // 次要色
  permission: string;   // 权限对话框边框色
  success: string;      // 成功/选中状态
  suggestion: string;   // 建议/聚焦状态
  inactive: string;     // 不活跃/dim状态
  claude: string;       // Claude品牌色
  error: string;        // 错误色
  warning: string;      // 警告色
  // ...
};
```

**颜色解析**（`ThemedText`）:
1. 显式 color prop 优先
2. TextHoverColorContext 次之
3. dimColor 最低优先级（使用 theme.inactive）

**原始颜色支持**: `rgb()`, `#hex`, `ansi256()`, `ansi:` 前缀自动识别为原始色值

### 4.2 布局和尺寸

- **Box**: Flexbox布局（`flexDirection`, `flexGrow`, `flexShrink`, `paddingX/Y`, `marginX/Y`）
- **终端尺寸**: `useTerminalSize()` 返回 `{rows, columns}`
- **高度约束**: 全屏模式下 `maxHeight` 限制
- **紧凑模式**: 终端宽度 < 120 时 FuzzyPicker 切换紧凑布局

### 4.3 特殊字符

| 字符 | 用途 |
|------|------|
| `❯` (figures.pointer) | 列表聚焦指示器 |
| `✓` (figures.tick) | 选中指示器 |
| `↓` `↑` (figures.arrowDown/Up) | 滚动指示器 |
| `●` (BLACK_CIRCLE) | Worker徽章圆点 |
| `▁▂▃▄▅▆▇█` | 语音模式波形光标 |
| `·` | 快捷键分隔符 |

### 4.4 边框样式

- **Pane**: 顶部彩色分隔线（`Divider`）
- **Dialog**: 通过 `Pane` 包装，`borderStyle="round"` 圆角边框
- **SearchBox**: 圆角边框（3行高）

### 4.5 响应式适配

- FuzzyPicker 动态计算 `visibleCount`: `Math.max(2, Math.min(requested, rows - CHROME_ROWS))`
- 紧凑模式：`columns < 120` 时缩短提示文本
- 预览位置：右侧需足够宽度，不足时切换到底部

## 5. 交互流程

### 5.1 用户输入到UI响应的完整流程

```
1. 用户在 TextInput 输入文字
   ↓
2. useTextInput hook 处理键盘事件
   ↓
3. BaseTextInput 更新 value/cursorOffset 状态
   ↓
4. React 组件重新渲染（React Compiler 自动跳过未变化子树）
   ↓
5. Ink 计算新布局
   ↓
6. ANSI转义序列发送到终端
   ↓
7. 终端更新显示
```

### 5.2 权限请求流程

```
1. 模型返回 tool_use 消息
   ↓
2. 权限系统检查是否允许（规则匹配/自动批准）
   ↓
3. 如需人工批准，渲染 PermissionRequest
   ↓
4. permissionComponentForTool() 分发到具体组件
   ↓
5. 显示工具特定UI（命令预览/文件diff等）
   ↓
6. 用户选择 Allow/Reject（键盘导航 + Enter）
   ↓
7. 可选输入反馈文字（Tab展开）
   ↓
8. 记录分析事件
   ↓
9. 回调 onAllow/onReject → 继续或拒绝工具执行
```

### 5.3 模糊搜索流程

```
1. 触发 FuzzyPicker（如 /model 命令）
   ↓
2. useSearchInput 捕获键盘输入
   ↓
3. onQueryChange(query) 通知父组件过滤
   ↓
4. 新 items 传入，setFocusedIndex(0) 重置焦点
   ↓
5. 窗口计算：windowStart = clamp(focusedIndex - visibleCount + 1, ...)
   ↓
6. 可见项渲染：items.slice(windowStart, windowStart + visibleCount)
   ↓
7. 焦点切换：step(delta) → setFocusedIndex
   ↓
8. Enter 选择 → onSelect(item)
```

### 5.4 主题切换流程

```
1. 用户通过 ThemePicker 选择主题
   ↓
2. ThemeProvider.setThemeSetting(newSetting)
   ↓
3. 如选 'auto'：setSystemTheme(getSystemThemeName()) 种子化
   ↓
4. 如选 'auto' 且 feature('AUTO_THEME')：启动 OSC 11 监听器
   ↓
5. useMemo 更新 context value
   ↓
6. 所有 useTheme() 消费者重新渲染
   ↓
7. ThemedText 重新解析颜色
```

### 5.5 状态更新机制

**AppState**（类Zustand模式）:
```typescript
const setAppState = useSetAppState();
setAppState(prev => ({
  ...prev,
  attribution: {
    ...prev.attribution,
    escapeCount: prev.attribution.escapeCount + 1
  }
}));
```

**选择器模式**: `useAppState(s => s.someField)` 仅在字段变化时触发重渲染

## 7. 远程模式UI

### 7.1 桥接模式（Bridge Mode）

**BridgeDialog** (`src/components/BridgeDialog.tsx`):
- 显示桥接连接状态和配置
- 包含连接URL、状态指示器
- 支持启动/停止桥接服务

### 7.2 远程环境

**RemoteCallout** (`src/components/RemoteCallout.tsx`):
- 远程模式下显示调用提示
- 标识当前在远程环境执行

**RemoteEnvironmentDialog** (`src/components/RemoteEnvironmentDialog.tsx`):
- 远程环境配置对话框
- 显示远程机器信息、连接状态

### 7.3 远程/桥接模式限制

1. **local-jsx 命令被禁止**: 在远程/桥接模式下，`local-jsx` 类型的命令不会渲染
2. **权限对话框简化**: 远程模式下权限交互可能通过API而非UI完成
3. **状态显示差异**: StatusLine 可能显示远程连接状态而非本地工具状态
4. **TextInput 行为**: 远程模式下输入可能转发到远程实例

### 7.4 组件组成树（远程模式）

```
<App>
├── ThemeProvider
│   └── FullscreenLayout
│       ├── ScrollBox
│       │   └── Messages（受限消息集）
│       ├── RemoteCallout                    # 远程调用标识
│       ├── StatusLine（含连接状态）
│       ├── TextInput（转发到远程）
│       └── BridgeDialog（模态时）
```
