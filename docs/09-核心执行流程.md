# 09 - 核心执行流程

本文档详细分析 Claude Code restored 项目（v999.0.0-restored）的完整执行流程，从用户输入到模型响应，涵盖命令解析、消息构建、API 调用、工具调用循环、流式输出和会话管理。

---

### 3.2 命令类型系统

```typescript
// 三种命令类型:
type Command =
  | { type: 'prompt',   getPromptForCommand(), ... }   // 扩展为文本 → 发给模型
  | { type: 'local',    action(), ... }                // 本地文本输出
  | { type: 'local-jsx', jsx(), ... }                  // 渲染 Ink UI

// 命令执行流程:
斜杠命令 (/xxx)
    │
    ▼
parseSlashCommand()
    │
    ├── 匹配内置命令 → getCommand(name)
    ├── 匹配技能命令 → skills/ 目录匹配
    ├── 匹配插件命令 → 插件注册表查找
    └── 匹配工作流命令 → workflows/ 匹配
    │
    ▼
命令类型分发:
    │
    ├── 'prompt' 类型:
    │   cmd.getPromptForCommand(args) → 返回文本
    │   → 文本作为 UserMessage 发给模型
    │   → 继续 query() 流程
    │
    ├── 'local' 类型:
    │   cmd.action(args, context) → 返回文本
    │   → 直接输出到终端
    │   → 不调用模型
    │
    └── 'local-jsx' 类型:
        cmd.jsx(args, context) → 返回 React 组件
        → 渲染到 TUI
        → 用户交互完成后返回
```

### 3.3 命令优先级

```
命令解析优先级 (高 → 低):
1. 精确匹配内置命令名
2. 精确匹配命令别名
3. 技能目录匹配 (skills/ 下的 .md 文件)
4. 插件技能匹配
5. 工作流脚本匹配
6. 模糊匹配/前缀匹配 (typeahead)
```

### 4.3 消息格式定义

```typescript
// 核心消息类型 (types/message.ts)
type Message =
  | UserMessage        // { type: 'user', message: { role: 'user', content: ContentBlockParam[] } }
  | AssistantMessage   // { type: 'assistant', message: { role: 'assistant', content: ... }, uuid, apiError? }
  | SystemMessage      // { type: 'system', subtype: 'error' | 'warning' | 'info', text: string }
  | AttachmentMessage  // { type: 'attachment', attachment: { type: 'edited_text_file' | ... } }
  | ProgressMessage    // { type: 'progress', ... }
  | ToolUseSummaryMessage  // { type: 'tool_use_summary', ... }
  | TombstoneMessage   // { type: 'tombstone', message: AssistantMessage } // 删除标记

// ContentBlock 类型:
type ContentBlock =
  | { type: 'text', text: string }
  | { type: 'tool_use', id: string, name: string, input: object }
  | { type: 'tool_result', tool_use_id: string, content: string, is_error?: boolean }
  | { type: 'image', source: { type: 'base64', media_type: string, data: string } }
  | { type: 'thinking', thinking: string, signature: string }
  | { type: 'redacted_thinking', data: string }
```

### 5.4 重试和错误处理

```
withRetry() 重试策略:
┌──────────────────────────────────────────────────────────────────┐
│ 最大重试次数: DEFAULT_MAX_RETRIES = 10 (可配置)                   │
│ 退避策略: 指数退避 + 随机抖动                                    │
│   delay = BASE_DELAY_MS * 2^(attempt-1) + jitter(0~25%)         │
│   BASE_DELAY_MS = 500ms                                          │
│   maxDelayMs = 32000ms                                           │
│                                                                  │
│ 可重试错误:                                                       │
│   ├─ 429 (Rate Limit)    → 非 Claude.ai 订阅用户可重试            │
│   ├─ 529 (Overloaded)    → 最多 3 次, 之后触发 fallback           │
│   ├─ 408 (Timeout)       → 始终可重试                            │
│   ├─ 409 (Lock Timeout)  → 始终可重试                            │
│   ├─ 401 (Unauthorized)  → 刷新 API Key 缓存后重试               │
│   ├─ 5xx (Server Error)  → 始终可重试                            │
│   └─ 连接错误 (ECONNRESET, EPIPE) → 禁用 keep-alive 后重试       │
│                                                                  │
│ 不可重试错误:                                                     │
│   ├─ 400 (Bad Request)   → 除 max_tokens 溢出外                  │
│   ├─ 403 (Forbidden)     → 除 token revoked 外                   │
│   └─ 4xx (其他客户端错误)  → 直接抛出                             │
│                                                                  │
│ 特殊处理:                                                         │
│   ├─ Fast Mode 冷却: 429/529 → 切换标准速度模型                   │
│   ├─ 529 Fallback: 连续 3 次 529 → 切换到 fallbackModel          │
│   ├─ OAuth 401: 强制刷新 token → 创建新客户端                     │
│   ├─ max_tokens 溢出: 解析错误 → 调整 maxOutputTokens → 重试      │
│   └─ 持续重试模式 (CLAUDE_CODE_UNATTENDED_RETRY):                │
│       无限重试, 最大退避 5 分钟, 心跳 30 秒                        │
└──────────────────────────────────────────────────────────────────┘
```

### 5.5 非流式降级

```
流式失败 → 非流式降级流程:
┌─────────────────────────────────────────────────────────────────┐
│ 1. 流式请求失败 (网络错误/超时/SDK 异常)                         │
│ 2. 记录 tengu_streaming_fallback_to_non_streaming 事件           │
│ 3. 切换到非流式模式:                                             │
│    ├─ messages.create({ stream: false, ... })                    │
│    ├─ 超时: API_TIMEOUT_MS (非流式最大 10 分钟)                  │
│    └─ max_tokens 限制: 64000                                     │

│ 5. 成功后继续正常流程                                           │
│ 6. 失败则向上抛出错误                                           │
│                                                                 │
│ 注意: 启用 tengu_disable_streaming_to_non_streaming_fallback     │
│ 时会跳过降级, 直接抛出错误 (防止工具重复执行)                     │
└─────────────────────────────────────────────────────────────────┘
```

```
│                                                                  │
│ 2. Context Collapse (上下文折叠):                                │
│    ├─ 触发: 消息数量过多                                         │
│    ├─ 策略: 将旧消息折叠为摘要                                   │
│    └─ 恢复: recoverFromOverflow() 释放折叠                       │
│                                                                  │
│ 3. AutoCompact (自动压缩):                                       │
│    ├─ 触发: token 计数超过阈值                                   │
│    ├─ 策略: 调用模型生成对话摘要                                 │
│    ├─ 配置: CLAUDE_CODE_DISABLE_AUTO_COMPACT=1 禁用              │
│    └─ 结果: 历史 → 摘要消息                                     │
│                                                                  │
│ 4. Reactive Compact (反应式压缩):                                │
│    ├─ 触发: 413 Prompt Too Long 错误                            │
│    ├─ 策略: 紧急压缩以恢复                                       │
│    └─ 降级: 先尝试 collapse drain, 再 reactive compact           │
│                                                                  │
│ 5. History Snip (历史裁剪):                                      │
│    ├─ 触发: 配置启用                                            │
│    └─ 策略: 裁剪中间历史, 保留首尾                               │
│                                                                  │
│ Token 计数: tokenCountWithEstimation()                           │
│   └─ 使用 usage.input_tokens + 估算                              │
└─────────────────────────────────────────────────────────────────┘
```

### 8.4 会话恢复

```
恢复入口点:
┌─────────────────────────────────────────────────────────────────┐
│ 1. --resume <uuid>: 恢复指定会话                                │
```

## 10. 关键数据转换

### 10.1 请求参数构建

```
用户输入 → API 请求:

1. 消息转换:
   UserMessage → { role: 'user', content: ContentBlockParam[] }
   AssistantMessage → { role: 'assistant', content: ContentBlockParam[] }
   tool_result → { role: 'user', content: [{ type: 'tool_result', ... }] }

2. 系统提示:
   string[] → 拼接为完整系统提示字符串

3. 工具 Schema:
   Tool[] → JSON Schema 数组
   ├─ name, description
   ├─ input_schema (JSON Schema)
   ├─ cache_control (prompt caching 标记)
   └─ defer_loading (动态工具, 可选)

4. 其他参数:
   model: string           ← 当前模型
   max_tokens: number      ← 最大输出 token
   thinking: { type, budget_tokens } ← 思考配置
   system: string          ← 系统提示
   tools: ToolInput[]      ← 工具定义
   stream: true            ← 流式模式
```

### 10.2 响应解析

```
SSE Event → 内部消息:

message_start:
  → 初始化 partialMessage, 记录 TTFT

content_block_start:
  → 根据 type 初始化对应的 content block

content_block_delta:
  → text_delta      → 累积到 text block, 实时更新 UI
  → input_json_delta→ 累积到 tool_use.input
  → thinking_delta  → 累积到 thinking block

message_delta (stop_reason):
  → 'end_turn'     → 完成, 构建完整 AssistantMessage
  → 'tool_use'     → needsFollowUp = true, 进入工具执行
  → 'max_tokens'   → 触发恢复逻辑

message_stop:
  → 流结束, 构建最终消息
```

## 12. 错误路径总览

```
┌─────────────────────────────────────────────────────────────────┐
│ 错误路径与恢复策略:                                              │
│                                                                  │
│ 1. 网络连接错误:                                                 │
│    ├─ 指数退避重试 (最多 10 次)                                  │
│    ├─ ECONNRESET → 禁用 keep-alive → 重试                       │
│    └─ 最终失败 → CannotRetryError → yield API 错误消息           │
│                                                                  │
│ 2. 认证错误:                                                     │
│    ├─ 401 → 刷新 API Key/OAuth → 新客户端 → 重试                 │
│    ├─ 403 revoked → 刷新 OAuth → 重试                            │
│    └─ 持续失败 → 提示用户重新登录                                 │
│                                                                  │
│ 3. 速率限制 (429/529):                                          │
│    ├─ Fast Mode → 冷却 → 标准速度                                │
│    ├─ 529 x3 → Fallback Model                                   │
│    ├─ Retry-After 头 → 等待指定时间                              │
│    └─ CLAUDE_CODE_UNATTENDED_RETRY → 无限重试                    │
│                                                                  │
│ 4. Prompt Too Long (413):                                       │
│    ├─ 先尝试 Context Collapse drain                              │
│    ├─ 再尝试 Reactive Compact                                   │
│    └─ 都失败 → 返回错误, executeStopFailureHooks                │
│                                                                  │
│ 5. Max Output Tokens:                                           │
│    ├─ 首次 → 升级到 64k (ESCALATED_MAX_TOKENS)                  │
│    ├─ 恢复消息 → 让模型续传 (最多 3 次)                          │
│    └─ 耗尽 → 返回错误                                           │
│                                                                  │
│ 6. 工具执行错误:                                                 │
│    ├─ 权限拒绝 → 用户重新编辑命令                                │
│    ├─ 执行失败 → tool_result with is_error=true                 │
│    └─ 超时 → 中断工具, 返回错误结果                              │
│                                                                  │
│ 7. 用户中断 (Ctrl+C):                                           │
│    ├─ abortController.abort('interrupt')                        │
│    ├─ 消耗剩余工具结果 (合成 tool_result)                         │
│    ├─ cleanupComputerUseAfterTurn()                             │
│    └─ yield 中断消息 → return { reason: 'aborted_streaming' }   │
│                                                                  │
│ 8. Bug/异常:                                                     │
│    ├─ query.ts catch → yieldMissingToolResultBlocks              │
│    ├─ yield API 错误消息                                         │
│    └─ logAntError (内部) / logError (外部)                      │
└─────────────────────────────────────────────────────────────────┘
```
