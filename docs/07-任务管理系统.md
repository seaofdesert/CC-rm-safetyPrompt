# 07-任务管理系统

## 1. 任务系统架构

### 1.1 核心接口

任务系统建立在统一的 `Task` 接口之上（`src/Task.ts`），所有具体任务类型都实现此接口：

```typescript
export type Task = {
  name: string
  type: TaskType
  kill(taskId: string, setAppState: SetAppState): Promise<void>
}
```

这是调度层的多态入口 —— `getTaskByType()` 返回对应类型的 `Task` 对象，唯一的 dispatch 方法是 `kill()`。spawn/render 从未被多态调用（已在 #22546 中移除），所有七种 `kill()` 实现都只使用 `setAppState`。

### 1.2 任务状态基类

```typescript
export type TaskStateBase = {
  id: string              // 任务唯一标识，带类型前缀
  type: TaskType          // 任务类型
  status: TaskStatus      // 任务状态
  description: string     // 任务描述
  toolUseId?: string      // 触发此任务的 tool_use ID
  startTime: number       // 创建时间戳
  endTime?: number        // 结束时间戳
  totalPausedMs?: number  // 暂停总时长
  outputFile: string      // 输出文件路径
  outputOffset: number    // 已读取偏移量（用于增量轮询）
  notified: boolean       // 是否已发送完成通知
}
```

### 1.3 状态枚举

**TaskStatus**（`src/Task.ts` 第15行）：
```
'pending' | 'running' | 'completed' | 'failed' | 'killed'
```

**TaskType**（`src/Task.ts` 第6行）：
```
'local_bash' | 'local_agent' | 'remote_agent' | 'in_process_teammate' | 'local_workflow' | 'monitor_mcp' | 'dream'
```

### 1.4 状态机

所有任务遵循统一的状态机：

```
                    ┌─────────┐
                    │ pending │  ← createTaskStateBase 初始状态
                    └────┬────┘
                         │ registerTask()
                         ▼
                    ┌─────────┐
          ┌────────►│ running │
          │         └────┬────┘
          │              │
     ┌────┴────┐    ┌────┴────────────┐
     │completed│    │     failed      │
     └─────────┘    └────────┬────────┘
          │                  │
          │         ┌────────┴────────┐
          └────────►│     killed      │
                    └────────┬────────┘
                             │
                    notified = true
                    evictAfter = now + PANEL_GRACE_MS (30s)
                             │
                             ▼
                    ┌─────────────┐
                    │  evicted    │  ← 从 AppState.tasks 中删除
                    └─────────────┘
```

**终端状态判定**（`src/Task.ts` 第27行）：
```typescript
export function isTerminalTaskStatus(status: TaskStatus): boolean {
  return status === 'completed' || status === 'failed' || status === 'killed'
}
```

### 1.6 中央状态注册表

所有任务存储在 `AppState.tasks: Record<string, TaskState>` 中，通过 `setAppState` 的不可变更新模式管理：

```typescript
// src/utils/task/framework.ts
export function updateTaskState<T extends TaskState>(
  taskId: string,
  setAppState: SetAppState,
  updater: (task: T) => T,
): void {
  setAppState(prev => {
    const task = prev.tasks?.[taskId] as T | undefined
    if (!task) return prev
    const updated = updater(task)
    if (updated === task) return prev  // 无变更检测，避免多余 re-render
    return { ...prev, tasks: { ...prev.tasks, [taskId]: updated } }
  })
}
```

### 1.7 驱逐（Eviction）模型

任务完成后不会立即从内存中移除，而是有一个宽限期：

| 常量 | 值 | 用途 |
|------|---|------|
| `POLL_INTERVAL_MS` | 1000ms | 任务轮询间隔 |
| `STOPPED_DISPLAY_MS` | 3000ms | 被停止任务在 UI 上的显示时长 |
| `PANEL_GRACE_MS` | 30000ms | 终端任务在面板中的保留时间 |

## 2. 任务类型详解

### 2.1 LocalShellTask（本地 Bash 命令）

**文件**：`src/tasks/LocalShellTask/LocalShellTask.tsx`

```typescript
type LocalShellTaskState = TaskStateBase & {
  type: 'local_bash'
  command: string                    // 要执行的命令
  result?: { code: number; interrupted: boolean }
  shellCommand: ShellCommand | null  // 底层进程句柄
  isBackgrounded: boolean
  agentId?: AgentId                  // 关联的 agent
  kind: 'bash' | 'monitor'           // UI 显示变体
  completionStatusSentInAttachment: boolean
  lastReportedTotalLines: number
  unregisterCleanup?: () => void
}
```

**核心功能**：
- `spawnShellTask()` —— 创建并启动后台 shell 任务
- `registerForeground()` —— 注册前台任务（可后续转为后台）
- `backgroundTask()` / `backgroundExistingForegroundTask()` —— 前后台切换
- `backgroundAll()` —— Ctrl+B 后台化所有前台任务

**停滞看门狗（Stall Watchdog）**：
```typescript
const STALL_CHECK_INTERVAL_MS = 5_000   // 每5秒检查一次
const STALL_THRESHOLD_MS = 45_000       // 45秒无增长视为停滞
const STALL_TAIL_BYTES = 1024           // 读取末尾1KB检测

// 检测交互式提示的正则模式
const PROMPT_PATTERNS = [
  /\(y\/n\)/i, /\[y\/n\]/i, /\(yes\/no\)/i,
  /\b(?:Do you|Would you|Shall I|Are you sure|Ready to)\b.*\? *$/i,
  /Press (any key|Enter)/i, /Continue\?/i, /Overwrite\?/i
]
```

当输出停止增长超过45秒且末尾匹配提示模式时，发送一次性通知，告知模型命令可能阻塞在交互式输入上。

**通知前缀**：`"Background command "`（用于 UI 折叠转换识别）

### 2.2 LocalAgentTask（本地后台 Agent）

**文件**：`src/tasks/LocalAgentTask/LocalAgentTask.tsx`

```typescript
type LocalAgentTaskState = TaskStateBase & {
  type: 'local_agent'
  agentId: string
  prompt: string                       // 发送给 agent 的提示
  selectedAgent?: AgentDefinition      // agent 定义
  agentType: string                    // 'general-purpose' 等
  model?: string
  abortController?: AbortController    // 用于取消
  unregisterCleanup?: () => void
  error?: string
  result?: AgentToolResult
  progress?: AgentProgress
  retrieved: boolean
  messages?: Message[]                 // 对话消息历史
  lastReportedToolCount: number        // 上次报告的 tool 使用数（用于增量）
  lastReportedTokenCount: number
  isBackgrounded: boolean
  pendingMessages: string[]            // SendMessage 队列的中间消息
  retain: boolean                      // UI 持有此任务：阻止驱逐
  diskLoaded: boolean                  // 已从 sidechain JSONL 加载
  evictAfter?: number                  // 面板可见性截止时间
}
```

**进度追踪器（ProgressTracker）**：
```typescript
type ProgressTracker = {
  toolUseCount: number
  latestInputTokens: number      // API 中 input_tokens 是累积的
  cumulativeOutputTokens: number // output_tokens 是每轮的，需累加
  recentActivities: ToolActivity[]  // 最多保留5条最近活动
}
```

**核心函数**：
- `registerAsyncAgent()` —— 创建后台 agent 任务（`isBackgrounded: true`）
- `registerAgentForeground()` —— 创建前台 agent 任务（`isBackgrounded: false`），返回 `backgroundSignal` Promise 用于中断
- `killAsyncAgent()` —— abort + 清理 + 设置 evictAfter
- `completeAgentTask()` / `failAgentTask()` —— 终端状态转换
- `updateAgentProgress()` / `updateAgentSummary()` —— 进度和摘要更新
- `backgroundAgentTask()` —— 前后台切换，解析 backgroundSignal
- `drainPendingMessages()` —— 在 tool 轮次边界排水中间消息

**AbortController 层级**：
```
Parent AbortController
  └── Child AbortController (自动随父级 abort)
        └── Agent 的执行逻辑
```

父级 abort 时子 agent 自动 abort，用于 InProcessTeammate 停止时级联停止子 agent。

## 3. 任务生命周期

### 3.1 完整创建与执行流程

#### 3.1.1 后台 Agent 任务（AgentTool → LocalAgentTask）

```
1. AgentTool 被模型调用
   │
2. generateTaskId('local_agent') → "a3k7x9m2"
   │
3. registerAsyncAgent({ agentId, description, prompt, ... })
   ├── initTaskOutputAsSymlink(agentId, agentTranscriptPath)  // 输出文件→转录文件符号链接
   ├── createChildAbortController(parentAbortController)       // 子 abort controller
   ├── createTaskStateBase(agentId, 'local_agent', desc, toolUseId)  // status: 'pending'
   │   └── 覆盖为 status: 'running', isBackgrounded: true
   ├── registerCleanup(async () => killAsyncAgent(agentId, setAppState))
   └── registerTask(taskState, setAppState)
       └── enqueueSdkEvent({ type: 'system', subtype: 'task_started', ... })
   │
4. Agent 执行 query() 循环
   ├── 每次 assistant 消息 → updateProgressFromMessage() → updateAgentProgress()
   ├── 中间用户消息 → queuePendingMessage() → 在 tool 轮次边界 drainPendingMessages()
   └── 进度更新时 emitTaskProgress()（SDK 消费者如 VS Code）
   │
5. Agent 完成
   ├── completeAgentTask(result, setAppState)
   │   ├── status: 'completed'
   │   ├── endTime: Date.now()
   │   ├── evictAfter: retain ? undefined : now + PANEL_GRACE_MS
   │   └── 清理 abortController, unregisterCleanup, selectedAgent
   ├── evictTaskOutput(taskId)  // 刷新并移除内存中的 DiskTaskOutput
   └── AgentTool 调用 enqueueAgentNotification()
       ├── 原子检查 notified 标志（防止重复通知）
       ├── abortSpeculation()  // 中止活跃推测
       └── enqueuePendingNotification({ value: <task_notification>..., mode: 'task-notification' })
   │
6. 下一轮 pollTasks() → generateTaskAttachments()
   └── 检测到 terminal + notified → evictTaskIds.push(taskId)
       → applyTaskOffsetsAndEvictions() → 从 AppState.tasks 中删除
```

#### 3.1.2 前台 Agent 任务

```
1. registerAgentForeground({ agentId, ..., autoBackgroundMs })
   ├── 创建 AbortController（非子级）
   ├── createTaskStateBase → status: 'running', isBackgrounded: false
   ├── 创建 backgroundSignal Promise
   │   └── backgroundSignalResolvers.set(agentId, resolve)
   ├── 如果 autoBackgroundMs > 0：设置 setTimeout 自动后台化
   └── registerTask() + 返回 { taskId, backgroundSignal, cancelAutoBackground }

2. Agent 执行 query()，在循环中 await backgroundSignal
   ├── 如果用户手动后台化 → backgroundAgentTask() → 解析 signal → agent 转为后台模式
   ├── 如果超时自动后台化 → timer 触发 → 设置 isBackgrounded: true → 解析 signal
   └── 如果 agent 在后台化前完成 → unregisterAgentForeground() → 直接从 tasks 中移除

3. 后台化后流程同后台 Agent
```

#### 3.1.3 Shell 任务（BashTool → LocalShellTask）

```
1. BashTool 被模型调用
   │
2. spawnShellTask({ command, description, shellCommand, ... }, context)
   ├── taskId 从 shellCommand.taskOutput 获取（TaskOutput 拥有数据）
   ├── createTaskStateBase → status: 'running', isBackgrounded: true
   ├── registerCleanup(async () => killTask(taskId, setAppState))
   ├── registerTask(taskState, setAppState)
   ├── shellCommand.background(taskId)  // 进程脱离 stdin，后台运行
   └── startStallWatchdog(taskId, ...)  // 启动停滞看门狗
   │
3. 进程执行中
   ├── 输出通过 TaskOutput 自动写入磁盘
   ├── 看门狗每5秒检查文件大小
   │   └── 45秒无增长 → tail 1KB → 匹配提示模式 → 发送通知
   └── shellCommand.result.then() 处理完成
       ├── cancelStallWatchdog()
       ├── flushAndCleanup(shellCommand)
       ├── 如果 status !== 'killed' → 根据 exit code 设置 completed/failed
       ├── enqueueShellNotification()
       └── evictTaskOutput(taskId)
```

### 3.2 状态转换函数汇总

| 转换 | LocalAgentTask | LocalShellTask | DreamTask | RemoteAgentTask |
|------|---------------|---------------|-----------|----------------|
| pending→running | `registerAsyncAgent()` 直接设为 running | `spawnShellTask()` 直接设为 running | `registerDreamTask()` 直接设为 running | `registerRemoteAgentTask()` 直接设为 running |
| running→completed | `completeAgentTask()` | `result.code === 0` | `completeDreamTask()` | 轮询检测到完成 |
| running→failed | `failAgentTask()` | `result.code !== 0` | `failDreamTask()` | 轮询检测失败 |
| running→killed | `killAsyncAgent()` | `killTask()` | `DreamTask.kill()` | `RemoteAgentTask.kill()` |

## 4. 任务委派

### 4.1 父子任务关系

任务通过 `AbortController` 层级和 `agentId` 关联实现委派：

```
主会话 (main-session)
  └── AgentTool 调用
       └── LocalAgentTask (a-prefix)
            ├── agentId 关联
            ├── ChildAbortController 级联取消
            └── 可进一步调用 AgentTool → 嵌套子 agent
```

### 4.2 子 Agent 创建链路

```
AgentTool (src/tools/AgentTool/AgentTool.tsx)
  → registerAsyncAgent({ agentId, parentAbortController, ... })
    → createChildAbortController(parentAbortController)
  → 子 agent 执行
    → 可调用 BashTool → spawnShellTask({ agentId: parentAgentId })
    → 可调用 AgentTool → 递归创建子 agent
```

### 4.3 孤离子任务清理

```typescript
// killShellTasksForAgent(agentId) —— 清理特定 agent 关联的所有 bash 任务
// 用于 agent 异常退出时清理可能遗留的 shell 进程
```

### 4.4 结果聚合

- Agent 完成后通过 `AgentToolResult` 返回结果给父级
- Shell 完成后通过 `result: { code, interrupted }` 返回
- 通知通过 `enqueuePendingNotification` 发送给模型，包含 `<result>`、`<usage>` 等 XML 标签

## 6. 任务队列

### 6.1 并发模型

任务系统 **不使用显式队列**，而是采用以下模型：

- **所有任务并行执行**：每个任务独立运行，没有并发限制
- **前台/后台模式**：通过 `isBackgrounded` 标志控制 UI 显示行为，不影响执行
- **AppState.tasks 注册表**：所有活跃任务存储在内存 Map 中

### 6.2 前台/后台模式

| 模式 | isBackgrounded | UI 行为 | 执行 |
|------|---------------|---------|------|

### 7.1 停止任务

```typescript
export class StopTaskError extends Error {

export async function stopTask(taskId: string, context): Promise<StopTaskResult> {
  // 1. 查找任务 → not_found
  // 2. 检查 status === 'running' → not_running
  // 3. getTaskByType(type) → unsupported_type（存根类型会触发）
  // 4. taskImpl.kill(taskId, setAppState)
  // 5. 对于 bash 任务：suppress exit code 137 通知，直接发送 SDK task_terminated
}
```

### 7.2 取消（AbortController）

```
ESC 键 → killAllRunningAgentTasks()  // 协调器模式下停止所有子 agent
Ctrl+C → 进程信号 → registerCleanup 回调触发 → kill()
stopTask() → taskImpl.kill()
```

每种 kill 实现：
- **LocalShellTask**：`killTask()` → `shellCommand.kill()` + 清除超时 + 设置 `'killed'`
- **LocalAgentTask**：`killAsyncAgent()` → `abortController.abort()` + 清理 + 设置 `'killed'`
- **RemoteAgentTask**：设置 `'killed'` → 归档远程会话 → 发送 SDK 事件
- **DreamTask**：`abortController.abort()` + `'killed'` + `rollbackConsolidationLock()`

### 7.3 通知去重

所有通知发送前都原子检查 `notified` 标志：

```typescript
let shouldEnqueue = false
updateTaskState(taskId, setAppState, task => {
  if (task.notified) return task  // 已通知（如被 TaskStopTool 停止），跳过

  return { ...task, notified: true }
})
if (!shouldEnqueue) return  // 防止重复通知
```
