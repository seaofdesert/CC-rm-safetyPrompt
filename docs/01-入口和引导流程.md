## 3. 入口点详解

项目有 **4 个独立入口点**，各自服务不同用途：

### 3.1 `src/bootstrap-entry.ts` — 主入口（开发/生产）

```typescript
// 1. 处理 CLAUDE_CODE_OVERRIDE_CWD 环境变量
if (process.env.CLAUDE_CODE_OVERRIDE_CWD) {
  const targetDir = process.env.CLAUDE_CODE_OVERRIDE_CWD
  try {
    const resolved = realpathSync(targetDir)
    process.chdir(resolved)
  } catch {
    process.chdir(targetDir)
  }
}

// 2. 初始化 MACRO 全局对象（版本信息等）
ensureBootstrapMacro()

// 3. 加载 CLI 入口
await import('./entrypoints/cli.tsx')
```

**作用**：这是 `npm run dev` / `npm run start` 的入口。它做了两件事：
1. 支持通过 `CLAUDE_CODE_OVERRIDE_CWD` 环境变量覆盖工作目录（在 `process.cwd()` 被任何模块读取之前）
2. 初始化 `MACRO` 全局对象，然后动态导入 `cli.tsx`

### 3.2 `src/entrypoints/cli.tsx` — CLI 分发器

这是整个 CLI 的**路由中枢**。文件顶部有零模块加载的快路径：

```typescript
// 顶层副作用：禁用 corepack auto-pinning
process.env.COREPACK_ENABLE_AUTO_PIN = '0'

// CCR 环境（容器）增大子进程堆内存至 8GB
if (process.env.CLAUDE_CODE_REMOTE === 'true') {
  process.env.NODE_OPTIONS = existing ? `${existing} --max-old-space-size=8192` : '--max-old-space-size=8192'
}

// A/B 测试基线 — 必须在 BashTool/AgentTool 导入前设置
if (feature('ABLATION_BASELINE') && process.env.CLAUDE_CODE_ABLATION_BASELINE) {
  for (const k of ['CLAUDE_CODE_SIMPLE', 'CLAUDE_CODE_DISABLE_THINKING', ...]) {
    process.env[k] ??= '1'
  }
}
```

`main()` 函数实现了**多快路径分发**模式：

```
cli.tsx main()
  ├── --version / -v / -V          → 直接输出 MACRO.VERSION, return (零导入)
  ├── --dump-system-prompt         → 启用配置, 输出系统提示, return
  ├── --claude-in-chrome-mcp       → 启动 Chrome MCP 服务, return
  ├── --chrome-native-host          → 启动 Chrome Native Host, return
  ├── --computer-use-mcp            → 启动 Computer Use MCP 服务, return
  ├── --daemon-worker=<kind>        → 运行 Daemon Worker, return
  ├── remote-control / rc / remote  → Bridge 模式主入口, return
  │   / sync / bridge
  ├── daemon [subcommand]           → Daemon 守护进程, return
  ├── ps / logs / attach / kill     → 后台会话管理, return
  │   / --bg / --background
  ├── new / list / reply            → 模板任务, return
  ├── environment-runner            → BYOC 环境运行器, return
  ├── self-hosted-runner            → 自托管运行器, return
  ├── --worktree --tmux             → tmux 工作树, return
  ├── --update / --upgrade          → 重定向到 update 子命令
  ├── --bare                        → 设置 CLAUDE_CODE_SIMPLE=1
  └── [default]                     → 加载 src/main.tsx, 进入完整 CLI
```

所有非快路径最终都落入：

```typescript
const { startCapturingEarlyInput } = await import('../utils/earlyInput.js')
startCapturingEarlyInput()
const { main: cliMain } = await import('../main.tsx')
await cliMain()
```

### 3.3 `src/entrypoints/mcp.ts` — MCP 服务入口

将 Claude Code 的工具集暴露为 MCP (Model Context Protocol) 服务：

```typescript
export async function startMCPServer(cwd: string, debug: boolean, verbose: boolean): Promise<void> {
  const server = new Server({ name: 'claude/tengu', version: MACRO.VERSION }, { capabilities: { tools: {} } })

  // ListTools: 将所有工具注册为 MCP tools
  server.setRequestHandler(ListToolsRequestSchema, async () => {
    const tools = getTools(toolPermissionContext)
    return { tools: await Promise.all(tools.map(async tool => ({
      description: await tool.prompt(...),
      inputSchema: zodToJsonSchema(tool.inputSchema),
      outputSchema: ...,
    })))}
  })

  // CallTool: 调用工具
  server.setRequestHandler(CallToolRequestSchema, async ({ params: { name, arguments: args } }) => {
    const tool = findToolByName(tools, name)
    const finalResult = await tool.call(args, toolUseContext, hasPermissionsToUseTool, ...)
    return { content: [{ type: 'text', text: finalResult }] }
  })

  // 通过 stdio 传输启动服务
  const transport = new StdioServerTransport()
  await server.connect(transport)
}
```

**MCP 命令**：目前只有 `review` 命令作为 MCP command 暴露。

### 3.8 `src/main.tsx` — 完整 CLI 主入口

这是整个交互式 CLI 的**核心入口**，包含：

- Commander.js CLI 框架设置
- 所有命令行选项定义
- 交互式 REPL 启动流程
- 远程模式、SSH 模式、Direct Connect 模式
- 会话恢复（resume/continue）
- 迁移系统
- 插件和技能加载

`main()` 函数位于第 585 行：

```typescript
export async function main() {
  profileCheckpoint('main_function_start')
  process.env.NoDefaultCurrentDirectoryInExePath = '1'  // Windows 安全
  initializeWarningHandler()

  // 特殊模式处理：
  // - cc:// URL (Direct Connect)
  // - --handle-uri (Deep Link)
  // - claude assistant (KAIROS)
  // - claude ssh (SSH_REMOTE)

  // 判断交互式/非交互式
  const isNonInteractive = hasPrintFlag || hasInitOnlyFlag || hasSdkUrl || !process.stdout.isTTY
  setIsInteractive(isInteractive)
  initializeEntrypoint(isNonInteractive)

  // 确定客户端类型 (cli, sdk-cli, sdk-typescript, claude-vscode, etc.)
  setClientType(clientType)

  eagerLoadSettings()  // 预加载设置标志
  await run()          // 执行 Commander 程序
}
```

`run()` 函数：
- 初始化 Commander.js 程序
- 注册 `preAction` hook 来执行 `init()`
- 定义主命令（交互式 REPL 或 `--print` 模式）
- 注册所有子命令（通过 `src/commands.ts`）

### 4.2 启动关键路径计时

```
时间线 (典型启动，毫秒级):

 0ms ─── bootstrap-entry.ts 执行
         ├── CLAUDE_CODE_OVERRIDE_CWD 处理
         └── ensureBootstrapMacro()

 ~1ms ── cli.tsx 顶层副作用执行
         ├── COREPACK_ENABLE_AUTO_PIN = '0'
         ├── NODE_OPTIONS 设置 (CCR 环境)
         └── ABLATION_BASELINE 设置

 ~2ms ── cli.tsx main() 快路径检查
         ├── --version 匹配? → 直接返回 (~0ms 总耗时)
         └── 无匹配 → 继续

 ~5ms ── startCapturingEarlyInput()
         开始捕获 TTY 输入，防止被后续 Ink 渲染吞掉
```

### 5.3 工作目录处理

```typescript
// bootstrap-entry.ts
// 1. CLAUDE_CODE_OVERRIDE_CWD 优先处理
if (process.env.CLAUDE_CODE_OVERRIDE_CWD) {
  const resolved = realpathSync(targetDir)  // 解析符号链接
  process.chdir(resolved)
}

// bootstrap/state.ts
// 2. 初始状态中解析 cwd
function getInitialState(): State {
  const overrideCwd = process.env.CLAUDE_CODE_OVERRIDE_CWD
  const rawCwd = overrideCwd || cwd()
  try {
    resolvedCwd = realpathSync(rawCwd).normalize('NFC')  // NFC 规范化
  } catch {
    resolvedCwd = rawCwd.normalize('NFC')  // 回退 (CloudStorage EPERM)
  }
  return { originalCwd: resolvedCwd, projectRoot: resolvedCwd, cwd: resolvedCwd, ... }
}

// main.tsx
// 3. main() 中设置 cwd 状态
setCwdState(cwd)  // 通过 Shell.ts 的 setCwd 同步
```

### 5.4 恢复/回退行为

作为恢复项目，存在以下特殊处理：

1. **MACRO 全局对象**：由 `ensureBootstrapMacro()` 提供默认值，构建时可被 `bun:bundle` 替换
   ```typescript
   const defaultMacro = {
     VERSION: pkg.version,      // "999.0.0-restored"
     BUILD_TIME: '',
     PACKAGE_URL: pkg.name,
     VERSION_CHANGELOG: '',
     ISSUES_EXPLAINER: 'file an issue at ...',
     FEEDBACK_CHANNEL: 'github',
   }
   ```

2. **SDK API 桩**：`agentSdkTypes.ts` 中所有 SDK 函数都是 `throw new Error('not implemented')` 桩

3. **本地 Shim 包**：替代 NAPI 原生模块
   - `color-diff-napi` — 颜色差异
   - `modifiers-napi` — 修饰符处理
   - `url-handler-napi` — URL 处理
   - `@ant/claude-for-chrome-mcp` — Chrome MCP 桥接

4. **dev-entry.ts**：开发环境下的恢复状态检查，扫描缺失的相对导入

## 7. 总结

### 7.1 架构特点

1. **分层启动**: `bootstrap-entry.ts` → `cli.tsx` (分发器) → `main.tsx` (核心) → `run()` (Commander) → REPL
2. **快路径优化**: 大量子命令通过 `feature()` 门控和参数匹配实现零/低模块加载的快路径
3. **动态导入**: 几乎所有非顶层依赖都通过 `await import()` 延迟加载，减少启动时间
4. **并行预热**: MDM 子进程和 Keychain 读取在模块加载阶段并行启动
5. **预连接**: Anthropic API 的 TCP+TLS 握手在 init() 后、首次 API 调用前预连接
6. **Memoization**: 命令列表、技能列表等使用 `lodash-es/memoize` 缓存，按 cwd 区分
7. **恢复项目特征**: SDK API 为桩实现，NAPI 模块被 shim 替代，MACRO 提供构建时可替换的默认值
